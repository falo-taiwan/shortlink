短網址表格版說明頁

v102 把 shortlink 維護從 JSON 視角，改成更接近日常使用的表格視角。

你可以把它理解成：`links.json` 仍然是資料契約，但維護時不必先看 raw JSON，而是先看表格、再排序、分頁、編輯、刪除，最後才匯出新的 JSON。

shortlink v102 表格操作版 GitHub Pages ready

v102 是做什麼的

v102 不是重做資料格式，而是重做操作體驗。它讓維護者先從表格看資料，再進行：

1. 排序：先把資料整理到你想看的順序。
2. 分頁：一頁固定筆數，避免資料一多就難找。
3. 編輯：點選某列後，在右側表單修改。
4. 新增：直接在上方表單新增新短網址。
5. 刪除：刪除前做確認，避免誤刪。

v101 與 v102 差異

1. `v101`：比較像工程底稿，適合教學與對照 JSON。
2. `v102`：比較像實際維護頁，適合日常操作。
3. `v101`：JSON 是主體畫面。
4. `v102`：表格是主體畫面，JSON 只保留成輔助預覽。

v102 工作方式

1. 開頁先嘗試讀取 repo root 的 `links.json`。
2. 成功後，把資料載入表格。
3. 你可以依 `slug` 或 `target URL` 排序。
4. 你可以翻頁找資料。
5. 你可以點選某列的 `編輯`，在右側修改內容。
6. 最後匯出新的 `links.json`，再人工覆蓋主站 root。

links.json 格式不變

v102 雖然換成表格介面，但資料契約不變，仍然沿用最簡字串對字串格式。

{
  "123": "https://www.google.com/",
  "demo": "https://force.formosa-ai.com",
  "force": "https://force.formosa-ai.com/about/",
  "qrcode": "https://falo-taiwan.github.io/tools/qrcode/falo-qrcode-v102.html",
  "shortlink": "https://falo-taiwan.github.io/tools/shortlink/falo-shortlink-json-editor-v101.html"
}

也就是說，UI 可以升級，但資料面先不改，這樣 router 和後續 GAS 路線都不會被破壞。

/

│   └─ /shortlink/

重點是：`links.json` 與 `404.html` 仍放在 repo root，而 editor 與 guide 放在 `tools/shortlink/`。

人工更新流程

1. 打開 `falo-shortlink-json-editor-v102.html`。
2. 自動讀取目前 `links.json`。
3. 在表格中排序、找資料、編輯、新增、刪除。
4. 匯出新的 `links.json`。
5. 人工覆蓋主站 repo root 的 `links.json`。
6. 提交並部署主站。

這一版的限制

1. 沒有自動寫回 GitHub。
2. 沒有 GAS、Google Sheet、權限控管。
3. 沒有複雜欄位，仍只維護 `slug` 與 `target URL`。
4. 沒有多語系與額外 SEO 強化，先以維護效率為主。

為什麼 v102 值得做

因為 `v101` 比較像教材底稿，而 `v102` 才比較像你平常會真的打開來維護資料的頁面。它不是改架構，而是把「維護體驗」往前推了一步。
